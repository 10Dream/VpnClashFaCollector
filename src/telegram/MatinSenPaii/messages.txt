***🔥***

قابلیت‌های جدید:
• دیتای آفلاین — ۷,۸۵۴ ریزالور شناخته‌شده ایرانی و ۱,۹۱۹ رنج CIDR داخل باینری embed شده، نیازی به دانلود رنج 
ها نیست
• دستور local با دو حالت:
  ◦ findns local -o resolvers.txt — خروجی ریزالورهای تأیید شده (نرخ موفقیت بالا)
  ◦ findns local -o candidates.txt --discover — تولید آی‌پی‌های کاندید از کل فضای آی‌پی ایران
• حالت‌های discover:
  ◦ --sample N — انتخاب تصادفی از هر سابنت (پیش‌فرض: ۱۰)
  ◦ --batch N --offset M — دسته‌بندی بدون تکرار
  ◦ --full — تمام ~11 میلیون آی‌پی
• fetch --local بروزرسانی شد — الان ۷,۸۰۰+ ریزالور شناخته‌شده رو merge می‌کنه (نه CIDR خام)

• رفع مشکل tunnel check (resolve/tunnel و doh/resolve/tunnel) که روی subdomain delegation همه resolverها فیل می‌شد

[185.51.200.0/24](http://185.51.200.0/24)

• فلگ --proxy-auth برای پروکسی SOCKS با احراز هویت در تست‌های e2e

### 🕒 2026-03-06 01:52:24 UTC
> ↪️ **Forwarded**

***🔍***
 خب اینم findns — اسکنر DNS Resolver برای تانل
ابزار findns یه اسکنر سریع و قدرتمنده که بین هزاران DNS resolver، بهترین‌ها رو برای تانل DNS (مثل DNSTT و Slipstream پیدا می‌کنه. UDP پورت 53  و هم DoH پورت 443 رو ساپورت می‌کنه. کافیه لیست resolver بگیری، اسکن بزنی و بهترین resolver رو تحویل بگیری! 
***✅***
***🔗***
 پایپلاین کامل: Ping → Resolve → NXDOMAIN → EDNS → Tunnel → E2E
***🛡️***
 تشخیص خودکار هایجک DNS
***📏***
 تست سایز EDNS برای سرعت بیشتر تانل
***🚇***
 تست واقعی end-to-end — واقعاً کلاینت DNSTT رو اجرا می‌کنه و اتصال رو تأیید می‌کنه
***⚡***
 و 50 worker موازی — هزاران resolver در چند دقیقه
***📋***
 خروجی JSON آماده استفاده مستقیم
***🌍***
 لیست داخلی 7,800+ resolver منطقه‌ای + دانلود خودکار لیست جهانی
***💻***
 پروژه: 
[github.com/SamNet-dev/findns](http://github.com/SamNet-dev/findns)
***📖***
 راهنمای کامل فارسی: 
[github.com/SamNet-dev/findns/blob/main/GUIDE.md](http://github.com/SamNet-dev/findns/blob/main/GUIDE.md)
***⬇️***
 دانلود باینری آماده (Linux/macOS/Windows): 
[github.com/SamNet-dev/findns/releases](http://github.com/SamNet-dev/findns/releases)

آموزش راه‌اندازی DNS Server با استفاده از DNSTT و کلاینت NetMod VPN 
پیش‌نیازها: برای اجرای DNSTT به یک سرور خارجی و یک دامنه نیاز دارید.
مرحله اول: تنظیمات DNS
١. ایجاد یک A رکورد:
در داشبورد اکانت کلاودفلر خودتون، یک A رکورد بسازید و اون رو به آی‌پی سرور خودتون متصل کنید.
مثال: 
[dns.example.com](http://dns.example.com/)
٢. ایجاد NS رکورد:
یک NS رکورد بسازید و اون رو به A رکورد قبلی متصل کنید.
مهم: نام NS رکورد باید تک حرفی باشه.
مثال: 
[a.example.com](http://a.example.com/)
مرحله دوم: نصب و اجرای سرویس DNSTT 
١. اجرای اسکریپت:
به سرور خودتون SSH بزنید و اسکریپت زیر رو اجرا کنید:
bash <(curl -Ls 
[https://raw.githubusercontent.com/bugfloyd/dnstt-deploy/main/dnstt-deploy.sh](https://raw.githubusercontent.com/bugfloyd/dnstt-deploy/main/dnstt-deploy.sh)
)
     
٢. وارد کردن اطلاعات:
زمانی که از شما خواسته شد نام دامنه رو وارد کنید، نام رکورد NS که ساختید رو وارد کنید.
در مثال ما: 
[a.example.com](http://a.example.com/)
در مرحله بعد مقدار MTU رو بدون تغییر بگذارید.
در مرحله آخر و انتخاب بین SOCKS و SSH، گزینه SSH رو انتخاب کنید.
٣. ذخیره کلید عمومی:
در پایان، یک کلید عمومی به شما داده می‌شه، که باید این کلید رو کپی و در جایی ذخیره کنید، چون برای اتصال به سرور بهش نیاز دارید.
مرحله سوم: استفاده در اندروید با استفاده از کلاینت NetMod VPN 
١. نصب اپلیکیشن NetMod VPN
می‌تونید آخرین نسخه رسمی اپ NetMod VPN رو که در کانال تلگرام توسعه دهنده منتشر شده، بدون نیاز به فیلترشکن از لینک زیر دانلود کنید. 
[https://uploadb.me/qagphdejj8ar/Version3.2.8.611-Universal.apk.html.html](https://uploadb.me/qagphdejj8ar/Version3.2.8.611-Universal.apk.html.html)
مهم: بعد از دانلود فایل، از طریق تغییر نام، پسوند html. رو از انتهای نام فایل حذف کنید، تا بتونید نصبش کنید.
٢. تنظیمات اپ برای اتصال:
در صفحه اصلی، بر روی آیکون + کلیک کنید و گزینه آخر DNSTT + SSH رو انتخاب کنید و تنظیمات رو مطابق مثال زیر انجام بدید:
Remarks: نام دل‌خواه 
DNS Address: یکی از آدرس‌های آی‌پی DNS 
Nameserver: نام رکورد NS
PublicKey: کلید عمومی که قبلاً کپی کردید
Username: نام کاربری برای اتصال SSH 
Password: رمز عبور برای اتصال SSH
در انتها تنظیمات رو ذخیره کنید و کلید اتصال رو بزنید.
